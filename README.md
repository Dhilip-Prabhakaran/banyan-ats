# Banyan ATS

An applicant tracking system built end to end — from business analysis to deployment — as a learning-in-public project.

## Why this project

I'm a Business Analyst with a teaching background, building this to learn the full SaaS product cycle hands-on: requirements, design, frontend, backend, database, and deployment. Every decision is documented, every feature starts as a written requirement.

## How this repo is organized

| Path | What's in it |
|------|--------------|
| `docs/ba/` | Business analysis artifacts — problem statement, personas, user stories, process flows |
| `docs/adr/` | Architecture Decision Records — every tech choice, the alternatives considered, and why |
| `docs/GLOSSARY.md` | Plain-language explanations of every concept used in this project |
| `docs/LEARNING-LOG.md` | Running log of what each development slice taught me |

## Status

**Phase 0 — Project setup.** BA phase begins next; no application code yet.

## Roadmap

1. **Phase 1 — Business analysis:** problem statement, personas, user stories, prioritization
2. **Phase 2 — Solution design:** data model, API contract, architecture decisions
3. **Phase 3 — Build in vertical slices:** each feature delivered end to end, deployed from the first slice
4. **Phase 4 — AI features:** applying Gen AI to the product (resume parsing, matching, and more)
